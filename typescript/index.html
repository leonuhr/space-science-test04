Stage 5: TypeScript

Core Concepts

  • Static Typing JavaScript lets a variable be a number, then a string. TypeScript forces you to define types upfront, preventing runtime errors.
  • Interfaces You can define the exact shape of data. If a function expects a User, passing an object missing a name property throws an error immediately.
  • Compilation Browsers can't run TypeScript natively. It must be compiled (transpiled) down to plain JavaScript before it hits the browser.
  • Developer Experience The biggest benefit isn't the final code—it's the autocomplete and error checking in your code editor while you write it.

Source Code 
// Define the exact shape of our component props
interface CardProps {
  title: string;
  description: string;
  initialCount?: number; // Optional
}

// Function must return void, takes specific types
function renderCard(props: CardProps): void {
  let count: number = props.initialCount || 0;
  
  // TypeScript knows btn is an HTMLButtonElement
  const btn = document.querySelector<HTMLButtonElement>('#action-btn')!;
  
  btn.addEventListener('click', (event: MouseEvent) => {
    count++;
    console.log(`Count is strictly a number: ${count}`);
    
    // If you tried to do: count = "hello"
    // TypeScript would throw an error before running!
  });
}

// Safe execution
renderCard({
  title: "Type Safety",
  description: "Catching bugs at compile time."
});
