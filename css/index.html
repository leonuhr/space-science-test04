Stage 2: CSS Added

Core Concepts: Presentation

  • The Cascade CSS stands for Cascading Style Sheets. Rules flow from the top down, and more specific rules override general ones.
  • Layout & Flexbox Using display: flex, we can align the input and button side-by-side, and perfectly center the task text with the checkbox.
  • The Box Model Every element is a rectangle: content → padding → border → margin. Notice how the input field and button now have breathing room.
  • Pseudo-classes & States :hover, :focus, :checked — CSS can respond to user interaction without any JavaScript (e.g. crossing out text when checked).
Hit "Disable CSS" to see the page revert to Stage 1. That jarring shift is what CSS does — it turns a raw document into a beautiful user interface. But notice: clicking "Add Task" still does nothing.
Source Code — style.css
/* Layout — aligning the input and button */
.todo-form {
  display: flex;
  gap: 0.5rem;
  margin-bottom: 1.5rem;
}

/* Glassmorphism — the visual layer */
.todo-app {
  background: rgba(30, 41, 59, 0.6);
  backdrop-filter: blur(12px);
  border: 1px solid rgba(255,255,255,0.05);
  border-radius: 1.5rem;
  padding: 2rem;
  box-shadow: 0 25px 50px -12px rgba(0,0,0,0.5);
}

/* Interactive states */
.btn-primary:hover {
  background: #2563eb;
  transform: translateY(-1px);
}

/* State styling based on DOM structure */
.todo-checkbox:checked + .todo-text {
  text-decoration: line-through;
  color: #64748b;
}
