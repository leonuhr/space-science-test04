Core Concepts

  • The Cascade CSS stands for Cascading Style Sheets. Rules flow from the top down, and more specific rules override general ones.
  • Selectors & Specificity .card targets a class. #id is more specific. Inline styles win everything. The browser calculates a score to decide which rule wins.
  • The Box Model Every element is a rectangle: content → padding → border → margin. Layouts are built by controlling these four layers.
  • Pseudo-classes & States :hover, :focus, :active — CSS can respond to user interaction without any JavaScript.
Hit "Disable CSS" to see the page revert to raw HTML. That jarring shift is what CSS does — it separates visual presentation from content structure entirely. Try it.
Source Code — style.css
/* Glassmorphism — the visual layer */
.card {
  background: rgba(30, 41, 59, 0.6);
  /* frosted glass: blurs content behind */
  backdrop-filter: blur(12px);
  border: 1px solid rgba(255,255,255,0.05);
  border-radius: 1rem;
  padding: 2.5rem;
  box-shadow: 0 10px 25px rgba(0,0,0,0.5);
  /* smooth transition — no JS needed */
  transition: transform 0.4s ease;
}

/* Pseudo-class: reacts to hover state */
.card:hover {
  transform: translateY(-10px);
}
