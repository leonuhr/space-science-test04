Stage 1: Pure HTML

Core Concepts

  • HTML = Structure, not appearance HTML tells the browser what things are — a heading, a paragraph, a button. It has zero say in how they look.
  • Tags & Elements An element is an opening tag, its content, and a closing tag: <h2>Title</h2>. Tags wrap content to give it meaning.
  • The DOM The browser parses HTML into a tree called the DOM (Document Object Model). CSS and JS work by reading and modifying this tree.
  • Semantic HTML Use elements for their meaning, not their default looks. <button> is keyboard-accessible; a <div> styled as a button is not.
Without HTML: There is no content, no structure, no web page. Even the most complex React apps compile down to HTML elements. HTML is the unbreakable foundation — every other technology is built on top of it.
Source Code
<!-- The "Card" Component (Raw HTML) -->
<div>

  <!-- Heading: communicates hierarchy -->
  <h2>Frontend Development</h2>

  <!-- Paragraph: a block of descriptive text -->
  <p>Building the user interface of the web.</p>

  <!-- Button: a semantic interactive element -->
  <button>Learn More</button>

</div>

<!-- Notice: no colors, no layout,
     no fonts. The browser applies its
     own default styles to everything. -->

↓ Raw browser defaults. No stylesheet applied.
